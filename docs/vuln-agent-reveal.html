課題

現状の脆弱性管理における課題

⚠️

単一ソースへの依存

SIDfm 1社のみに依存。カバー範囲外の脆弱性を見逃すリスクがある。

🤚

手作業のボトルネック

メール確認 → SBOM照合 → 担当者連絡 → 記録。すべてが手動で対応遅延。

📢

通知のノイズ

自社に無関係な脆弱性も全件通知。重要な情報が埋もれる。

💸

固定コスト

年間ライセンス費用が継続発生。利用量に関わらず固定費。

カバレッジ

11の情報ソース

世界の公的データベースとベンダー公式APIを網羅的に監視。

公的データベース (4種)

  • NIST NVD — 米国標準の脆弱性DB (世界最大)
  • CISA KEV — 実際に悪用されている脆弱性一覧
  • JVN — 日本向け脆弱性情報
  • OSV — Google運営。OSS特化

ベンダー公式 (4種)

  • Cisco — ネットワーク機器
  • Microsoft — Windows / Office 等
  • Fortinet — ファイアウォール / VPN
  • AlmaLinux — サーバーOS

社内利用製品 (3種)

  • Zabbix — 監視ツール
  • MOTEX / LanScope — IT資産管理
  • SKYSEA Client View — 端末管理

運用

Google Chat との統合

通知を受け取るのも、分析を依頼するのも、すべてGoogle Chat上で完結。

🔔 自動通知を受け取る

新しい脆弱性が自社に関係する場合、AIが自動でChatに通知カードを投稿。

  • 何の脆弱性か (CVE番号・概要)
  • どのシステムに影響するか
  • いつまでに対応すべきか
  • どう対処すべきか

💭 AIに質問する

通知を見て詳しく知りたいときは、@メンションするだけ。

  • 「この脆弱性の影響範囲は？」
  • 「過去に同様の対応はあった？」
  • 「起票用のテキストを作って」
  • 「このCVEの最新情報を教えて」

🎯 既存運用との互換

SIDfm通知がChatに届くフローもそのまま活用可能。

  • SIDfmメール → Chat転送
  • ？リアクションで自動分析
  • 起票コピペ文を自動生成
  • 段階的に移行可能

重複排除

複数ソースが同じ脆弱性を報告しても、1回だけ通知。二重対応を防止。

ノイズ排除

自社のソフトウェア台帳 (SBOM) に載っていなければ通知しない。

全件記録

通知した/しないに関わらず、全脆弱性情報をデータベースに自動記録。
