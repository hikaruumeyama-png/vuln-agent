# ブラウザ拡張: サイドパネルAIチャット（開発予定）

このドキュメントは、将来実装するブラウザ拡張機能
「サイドパネルAIチャット」のバックログです。
現時点では未実装です。

## 目的
- 現在タブの URL と選択テキストを文脈として AI エージェントへ渡す。
- 「このページ内容を根拠に回答して」のような連続対話をしやすくする。
- 調査中の画面を離れず、右側サイドパネルで問い合わせを完結させる。

## MVP要件（実装対象）
1. サイドパネルUI
- チャット入力欄、送信ボタン、会話履歴表示。
- 履歴は1タブ単位で保持（タブを閉じると破棄）。

2. 文脈取り込み
- 「現在タブURLを添付」トグル。
- 「選択テキストを添付」ボタン（未選択時は無効化）。
- 送信時に `url` / `selected_text` / `user_query` をまとめて送る。

3. エージェント連携
- 既存 Gateway の WebSocket (`/ws`) または HTTP 経由で問い合わせ。
- エラー時は再試行導線と原因表示（認証切れ/接続失敗/タイムアウト）。

4. 連続対話
- 直近N件（例: 10件）の会話をコンテキストとして同送。
- 「このページを根拠に」の固定プロンプト挿入ボタン。

5. 最低限の安全制御
- 送信前にドメイン表示（どのページ情報を送るか明示）。
- オン/オフ可能な「ページ情報送信を許可」設定。

## 技術方針
1. 拡張仕様
- Manifest V3。
- `side_panel` + `storage` + `activeTab` + `scripting` + `tabs` 権限を最小採用。

2. 構成
- `background/service worker`: タブ情報取得・Gateway通信管理。
- `sidepanel`: チャットUI。
- `content script`（必要時のみ注入）: 選択テキスト取得。

3. 認証連携
- Gateway側 OIDC セッション（既存）を前提にする。
- 拡張は原則トークン保持しない（Cookieセッション利用）。
- 未認証時は `/auth/login` を開く導線のみ提供。

4. 通信形式
- 初期はテキスト問い合わせのみ（音声なし）。
- 送信payloadの標準形:
```json
{
  "type": "user_text",
  "text": "ユーザー質問",
  "context": {
    "page_url": "https://example.com/post/1",
    "selected_text": "選択部分",
    "conversation_tail": ["..."]
  }
}
```

## 受け入れ基準（Done）
1. サイドパネルから質問送信し、応答が表示される。
2. URL添付ON時、現在タブURLがコンテキストに含まれる。
3. 選択テキスト添付時、選択文がコンテキストに含まれる。
4. 同一タブでの連続対話時、直前履歴が参照される。
5. 未認証時にログイン導線が表示され、認証後に再送できる。
6. ページ情報送信OFF時、URL/選択テキストを送らない。

## 非目標（MVP外）
- 音声対話の組み込み。
- ページ全体DOMの自動スクレイピング送信。
- 複数Gatewayの同時接続。
- 高度なRAGインデックス構築。

## 想定リスク
- 機密情報を含むページ内容の送信リスク。
- サイト権限の過剰要求による審査・運用コスト増。
- タブ単位履歴と全体履歴の混同による誤文脈。

## 次フェーズ候補
1. 参照元リンク付き回答（どのページ文脈を使ったか明示）。
2. ページ内 CVE/PURL 自動抽出チップ。
3. SBOM照合ショートカット（選択語を直接検索）。
